# 臺灣人文藝術資料庫 — 計畫書

> 2026-07-18 立案（David 拍板：先做人物試點）。本檔回答四件事：這個資料庫的脈絡是什麼、內容從哪裡來、怎麼寫怎麼擴、跟 Taiwan.md 的分工。
> 部署與技術細節見 `docs/DEPLOY.md`；本檔只談內容與路線。

## 一、定位與脈絡

這個資料庫跟「認識臺灣」（taiwan-geo-db）是成對的兩半：

- **認識臺灣**管「土地」——34 縣市頁＋19 議題頁，從空間認識臺灣。
- **臺灣人文藝術**管「人」——從人物、作品、文化實踐認識臺灣，時間軸走的是人的一生與時代。

兩者共同服務同一個教學主張：立足臺灣的博雅人文素養——先認識自己、再連結世界。使用場景有兩個：David 的五年級人文主課程備課（華德福高年級以傳記與敘事進入歷史，「從一個人看一個時代」），以及「人與社會」教師社團的共享資源（社團站首頁掛三個資料庫連結）。

讀者設定是「正在備課的老師」，每頁的終點是課堂：主文寫成可以直接改編成說書的敘事，附教學素材與說書切分提示。

## 二、與 Taiwan.md 的關係：意義會不會重疊？

不會，前提是分工清楚。Taiwan.md 已經是一個 851 篇、有腳註紀律、每天演化的臺灣知識庫，它的讀者是世界與 AI；再做一個小號的它沒有意義。我們的意義在它不做的那一層：

| 層         | 誰做        | 內容                         |
| --------- | --------- | -------------------------- |
| 百科事實層     | Taiwan.md | 有腳註的事實與敘事、多語投影、給大眾與 AI 讀   |
| **教學轉化層** | **本資料庫**  | 適齡改寫、說書切分、課堂活動、地理錨點、與主課程串接 |

**代換規則（一頁人物要上架時照這三條走）**：

1. **Taiwan.md 有專文** → 取材改寫＋逐條引用回鏈（CC BY-SA 標示）。試點六位全屬這類。
2. **Taiwan.md 沒有專文**（洪通、陳達、黃土水、李天祿、廖瓊枝……都查過沒有）→ 我們自己寫、自己找一手來源。寫成熟後，可經 David 的 fork（elliot200852-lab/taiwan-md）把「百科事實層」的部分貢獻回上游——授權同為 CC BY-SA，雙向相容。教學層（說書、活動）留在本庫，上游不收教案。
3. **純教學件**（說書稿、切分、活動設計）→ 永遠只在本庫，這是我們的本體。

一句話：我們是 Taiwan.md 生態的**教育端**，取材於它、回饋給它，但不復刻它。

## 三、呼應設計

### 呼應 Taiwan.md：三層深度＋腳註紀律

Taiwan.md 每篇文章分三層（30 秒→5 分鐘→全文）、每 300 字至少一條腳註。本庫的人物頁結構刻意同構：

- 「他是誰」＝30 秒層
- 「生平與時代」＋「作品與聽看入口」＝5 分鐘層（說書底稿）
- 「出處」編號腳註＝深查層，每條回鏈 Taiwan.md 或其原始來源

寫作也沿用它 EDITORIAL 的反空話紀律（他們叫「塑膠偵測」）：禁百科腔開頭、禁對立翻轉句式、事實必有出處。

### 呼應地理：人物×地方雙鏈

每個人都長在一個地方。每頁人物固定帶一個**地理錨點**區塊，直連認識臺灣的縣市頁：

| 人物 | 錨點 | 連到 |
|---|---|---|
| 陳澄波 | 嘉義市 | taiwan-geo-db `pages/chiayi-city.html` |
| 李梅樹 | 新北三峽 | `pages/new-taipei.html` |
| 鄧雨賢 | 桃園龍潭（客庄） | `pages/taoyuan.html` |
| 鄧麗君 | 雲林褒忠 | `pages/yunlin.html` |
| 林懷民 | 嘉義新港 | `pages/chiayi-county.html` |
| 侯孝賢 | 高雄鳳山（成長地） | `pages/kaohsiung.html` |

課堂效果：說完陳澄波，隔天地理課打開嘉義市頁，孩子看見同一座城的山水與街市——人與地在課程裡互相佐證。反向鏈（縣市頁列「這裡的人物」）排在路線圖 Phase 3，等人物量夠了再回鏈，避免現在就去動 geo-db 的 build。

### 領域標籤：人物頁 chips → 首頁 #field-\<tag\> 篩選

2026-07-18 加：每位人物 frontmatter 帶 `tags`（第一個為主要領域），人物頁 hero 顯示可點 chips，連回首頁 `#field-<tag>`；首頁人物 tab 上方同步生成「全部＋出現過的領域」篩選列，hash 路由與既有 `#general`/`#people` 共存。這是分領域完整頁面上線前的雛形，URL 與資料結構刻意設計成可無痛升級——Phase 2 分領域頁上線後，同一批標籤與 chip 原樣承接，不必重寫。

## 四、內容來源（層級由高到低）

1. **Taiwan.md**（本機 `taiwanmd` CLI：search／read／cite；知識庫快取在 `~/.taiwanmd`）——首選，因為有腳註紀律。
2. **Taiwan.md 文章引用的原始來源**——寫腳註時優先追到一手（館方、報導、研究）。
3. **公立館所與官方資料庫**：國家文化記憶庫、開放博物館、國美館／北美館典藏查詢、臺灣音聲一百年（史料館）、國家電影及視聽文化中心——用於 Taiwan.md 沒有的人物與作品細節。
4. **Wikimedia Commons**——肖像與作品圖，只用授權明確（PD／CC BY／CC BY-SA）者，頁尾逐張標示。
5. 縣市文化局、基金會（陳澄波文化基金會、李梅樹紀念館等）官網——展覽與參觀資訊。

紀律：不確定的年份與說法寧可不寫；每頁上架前抽查 2 條腳註是否真能支撐主張（Taiwan.md 內容大量由 AI 生成再審核，引用前要驗）。

## 五、怎麼寫：單頁生產管線

```
選人（課程需要優先）
  → taiwanmd search/read/cite 取材（無專文則走來源 3、4）
  → 寫 content/people/<slug>.md（主文 600–900 字敘事＋教學素材＋說書切分＋腳註；
    格式照現有六篇，區塊對應 templates/person.html；本機用 Obsidian 編）
  → 抽查腳註 → 肖像找 Commons（授權不明就不放圖）
  → python3 scripts/build_pages.py（content/ → _build/，鏡射 Drive html 夾結構）
  → gws 上傳 _build/ 到 Drive html 夾（圖另傳 img 夾）
  → CI 部署 → scripts/verify_live.py 全綠才算完成
```

派工紀律照 MyWork harness：批次生產每批 2 支 agent（一支 3 頁上下）、產出者不自驗、David 抽驗文字。單篇也可以反過來：David 口述他對這個人的觀點，AI 整理成頁——他的話優先於百科。
