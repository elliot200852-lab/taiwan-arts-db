## 0. 一頁摘要（給趕時間的 agent）

- 你在做的是：臺灣歌曲百年史，八個時代頁＋一本歌曲登記簿（`content/songs/era-*.yaml` 分片），掛在首頁「臺灣歌曲」tab。
- 主軸只有一個：**傳統與變革**——每頁都要讓讀者看見「這個時代的歌從哪裡繼承、被什麼改變」。
- 兩條產品鐵則（David 2026-07-18 拍板）：**歌名即連結**——不論在歌單區或正文，點歌名就直達 YouTube（§2.1、§6 的自動掛鏈機制）；**版本以原唱優先**——找不到原唱的合格來源，才用有名的翻唱（§5）。
- 三條紅線，踩到就是打掉重做：
  1. **歌詞零引用**（歌名以外，任何歌詞原文一個字都不准出現在任何產出裡）。
  2. **連結只走登記簿**，且每條符合 §5 授權優先序、通過 `scripts/check_songs.py` 驗證。
  3. **事實必有出處**，腳註格式照現有 field 頁；查不到可靠出處的說法寧可不寫。
- 你不可以做的事在 §8「不做清單」——先讀完再動手。

### 2.1 時代頁 `content/songs/era-<slug>.md`

格式同 `content/fields/*.md`（frontmatter＋受限 Markdown＋出處清單），差異只有 frontmatter 多兩欄：

```yaml
---
title: 曲盤時代            # 期名，照 §1 表
slug: era-78rpm
period: 1932–1945          # 年代，照 §1 表
order: 2                   # 時代序，首頁 tab 依此排
axis: 唱片工業讓歌第一次可以被反覆重播；戰爭末期歌被改成時局歌曲。   # 主軸一句話（首頁時代卡用；照 §1 表意旨，可微調更口語）
---
```

正文規格：

- **700–1100 字**敘事，2–4 個 `##` 小節，每節圍繞「傳統與變革」主軸推進——開頭直接進場景或人物（禁百科腔「XX 是指⋯」開頭），結尾接到下一個時代的門口。
- 人名凡有人物頁者一律 `[名](../pages/<slug>.html)` 連結；地名凡 geo-db 有頁者連 geo-db（絕對 URL，照 field 頁現例）。
- 歌名在正文用《》書名號。**正文不手寫任何試聽 URL**——build 會把正文中出現在登記簿裡的《歌名》自動掛上該歌的主連結（`listen[0]`，見 §6），做到「點歌名直達 YouTube」而且全站只有登記簿一個連結來源。寫完文章要自查：正文點到名的每一首歌，都必須已經在登記簿裡（跑 `check_songs.py` 會列出「正文有《》但登記簿查無」的孤兒歌名，一首都不准留——沒有連結的歌名就是沒交完的稿）。
- 腳註紀律照現有頁：`[^N]` 上標＋文末「## 出處」有序清單，每條寫清楚支撐了哪些主張＋可點 URL。每頁至少 4 條。
- 寫作禁令（與全庫一致）：禁對立翻轉句式（「不是A而是B」「與其說⋯不如說」「不僅是⋯更是」）、禁機械排比與金句腔、事實不確定寧可不寫。

### 2.2 歌曲登記簿 `content/songs/era-<slug>.yaml`（分片）

每片頂層一個 `songs:` 清單，**每首歌一筆**，欄位如下。標｜必｜的缺了 build 直接 fail；片內歌曲的 `era` 必須等於檔名的 slug：

```yaml
songs:
  - id: bang-chhun-hong          # ｜必｜唯一 slug：臺語歌用臺羅、國語歌用漢語拼音、族語歌用該語羅馬字，全小寫連字號
    title: 望春風                 # ｜必｜通行漢字歌名
    year: 1933                   # ｜必｜首次發表年（整數；考證不確定寫概數年並在 note 說明）
    era: era-78rpm               # ｜必｜§1 表內的 slug，僅此 8 值
    language: 台語                # ｜必｜台語｜國語｜客語｜族語（阿美）等——族語括號註明族別｜日語｜混合
    credits:                     # ｜必｜至少一欄；查證不到的欄位省略、不猜
      lyricist: 李臨秋
      composer: 鄧雨賢
      original_singer: 純純
    people: [deng-yu-hsien]      # 選填：已有人物頁者的 slug 清單（build 會驗 slug 存在）
    hook: 二十四歲的李臨秋寫下少女心事，譜成臺灣最常被翻唱的旋律。   # ｜必｜≤45 字一句話，不得含歌詞
    listen:                      # ｜必｜至少 1 條、至多 3 條試聽連結；listen[0]＝主連結（歌名點下去的那條）
      - url: https://www.youtube.com/watch?v=XXXXXXX
        label: 純純 1933 原版（國立臺灣歷史博物館・臺灣音聲一百年）
        source_type: archive     # official｜archive｜foundation｜broadcaster｜other（§5 定義）
        version: original        # ｜必｜original（原唱）｜famous_cover（有名翻唱，label 寫明誰唱）｜live（現場）
    banned: 戒嚴時期一度遭禁（理由簡述＋出處編入 sources）    # 選填：禁歌史，是「傳統與變革」的重要教材
    note: ""                     # 選填：年代考證、版本說明等一句話補充
    sources:                     # ｜必｜至少 1 條——本筆事實（年份、詞曲、禁歌史）的出處，格式「說明：URL」
      - 詞曲作者與 1933 年發表：https://zh.wikipedia.org/zh-tw/望春風
```

規模紀律（David 2026-07-18 校準：**數量要多，且要有代表性**）：全線目標 **250–350 首**、單一 era **20–50 首**，上不封頂——但「多」靠的是把有代表性的歌收齊，不是灌水。每首入簿要過兩關：**代表性關**（傳唱度、歷史意義、教學價值至少居其一，選歌理由一句話記進研究筆記）＋**驗證關**（連結與出處照本檔）。同名湊數、無法通過連結關的歌，寧缺勿濫。

## 3. 寫作紀律（時代頁與 hook 共用）

1. **歌詞零引用（紅線）**：歌名以外，歌詞原文一個字都不出現——hook、正文、label、note、研究筆記全部適用。描述歌在唱什麼，用自己的話轉述（「唱的是少女望著心上人的心事」可以；引原句不可以）。
2. **適齡**：讀者是備課老師，最終聽眾是五年級孩子。禁歌、政治事件照寫（這正是主軸），但行文不渲染暴力細節。
3. **敘事優先，要有人文氣息與歷史感**（David 2026-07-18 拍板）：每個時代從一首歌或一個人的具體場景進場，時代背景為敘事服務；不寫成編年流水帳。每頁**穿插 1–3 則有趣的真實小故事**（歌的誕生軼事、被禁的荒謬理由、一句話改變一首歌命運的時刻），小故事必須查證屬實、有出處入腳註——趣味來自史實本身，不添油加醋。
4. **只寫查證得到的**：年份、首唱者、禁歌原因這類硬事實，兩個獨立來源對不上就降級處理（寫進 note 標示存疑，或不寫）。研究筆記要留下「查了什麼、為何取捨」。

## 5. 試聽連結治理（登記簿 `listen` 欄）

**版本優先序（David 2026-07-18 拍板，決定 `listen[0]` 主連結——歌名點下去聽到誰）**：

1. **原唱／歷史原版**（`version: original`）——儘量找原唱，這是預設。
2. 原唱在下方授權優先序 1–4 內都找不到 → **有名的翻唱**（`version: famous_cover`；「有名」＝歌手本身查得到、版本有傳唱度，label 寫明誰唱的）。
3. 兩者都找不到合格來源 → 這首歌不入簿（或改收官方現場版 `version: live`），研究筆記記一筆。

原唱與著名翻唱都找得到時，鼓勵兩條都收：`listen[0]`＝原唱主連結、`listen[1]`＝著名翻唱——老歌尤其有教學價值（同一首歌兩個時代的聲音，正是「傳統與變革」）。

**授權優先序**（在上述版本優先序內逐層找來源，高者優先）：

1. `official`——藝人／唱片公司官方頻道（YouTube 官方認證頻道、官方 Topic 頻道）。
2. `archive`——公立典藏機構頻道或站內播放頁（臺灣音聲一百年、國影中心、文化記憶庫等）。
3. `broadcaster`——公廣集團與主流媒體官方頻道（公視、央廣、大愛、各大電視台官方上傳之節目片段）。
4. `foundation`——基金會、紀念館、政府文化單位官方帳號。
5. `other`——以上皆無時，才可收「上傳者身分可辨識且顯非侵權」的來源（如學術機構、音樂節官方）。**個人帳號上傳的完整歌曲一律不收**——原唱只剩個人上傳時，照版本優先序第 2 條改找有名翻唱的合格來源，不要為了原唱破授權例。

硬規則：

- 連結必須直達單曲（YouTube 單支影片、典藏單頁），不收歌單頁、搜尋結果頁、首頁。
- 每條 `label` 寫明「誰的版本＋來源機構」，讓老師課前不用點開就知道會放出什麼。
- 不內嵌播放器、不下載音檔、不轉存任何音訊——本庫只放連結。
- 驗證：`check_songs.py` 對 YouTube 連結打 oEmbed（`https://www.youtube.com/oembed?url=…`，404＝影片不存在即 fail）、其他網址驗 HTTP 200。**通過驗證只代表連結活著，不代表來源合格——來源合格性由盲驗 agent 逐條人工判**（§7）。

## 6. Build 與頁面整合（S1 基建，一次做完後續批次不再動）

- 首頁新 tab「臺灣歌曲」：hash `#songs`，TABS 加 `songs`；panel 內容＝八張時代卡（期名＋年代＋frontmatter `axis` 主軸句，依 `order` 排），連到 `pages/song-<slug>.html`（slug 已含 era- 前綴，如 `pages/song-era-78rpm.html`）。hash 路由照現有模式擴充，不動既有 tab 行為。分片與 MD 一片都還沒有時顯示占位文字不 crash（同地圖 tab 過渡態前例）。
- 時代頁 `pages/song-<slug>.html`：hero（期名＋年代）→ 正文（era MD 轉出）→ **「這個時代的歌」歌單區（build 讀入全部登記簿分片、取該 era、依 year 排序自動生成**：歌名＋年份＋語言＋詞曲唱＋hook＋listen 連結＋禁歌標記；歌單不手寫，登記簿是唯一來源）→ 出處。頁尾加上一期／下一期導覽。
- **歌名即連結（兩處都是）**：歌單區的歌名本身＝`listen[0]` 主連結（`target="_blank" rel="noopener"`，直達 YouTube／典藏單頁）；`listen[1..]` 以「另聽：誰的版本」小字列於後。正文轉換時，build 掃描《歌名》——凡與登記簿 `title` 完全比對命中者，自動包成 `listen[0]` 連結；未命中者保留純文字並由 `check_songs.py` 回報孤兒歌名清單（讓寫手補登記簿，不讓正文出現點不了的歌名）。同名異曲用登記簿 `note` 標注、正文改寫成可區辨的寫法後人工指定，不硬猜。
- `scripts/check_songs.py`：schema 全欄驗證（必填、era 合法值、id 唯一、people slug 存在、hook 長度）＋連結驗證（§5）＋「每 era 至少 1 首才可出頁」。`build_pages.py` 開頭呼叫，驗不過整個 build fail-fast。
- 部署鏈不變：build → gws 上傳 Drive html 夾 → CI → `verify_live.py`（歌單區的外部連結由 check_songs 管，verify_live 照舊管站內完整性）。

## 7. 派工與驗收（照 MyWork harness，逐條執行）

- **批次單位**：一批 2 支 agent——A 支「寫」（1 個時代頁＋該期登記簿條目＋研究筆記），B 支「盲驗」（fresh context，不看 A 的過程只看產出）。禁一歌一支 agent。
- **盲驗 checklist**（B 支逐條回報 PASS/FAIL＋證據）：
  1. 歌詞零引用掃描（含 hook、label、note）。
  2. 逐條點開 listen 連結：影片存在、內容確為該曲、來源符合 §5 授權優先序與 source_type 標示、**版本照原唱優先序**（標 original 的真是原唱；用 famous_cover 的，回報「為何找不到原唱合格來源」一句話）。
  2b. 抽建置後的時代頁實測：歌單區與正文各抽 5 個歌名，點下去直達正確的 YouTube 影片；正文無孤兒歌名（check_songs 清單為空）。
  3. 抽 3 條腳註／sources 驗證真能支撐主張。
  4. 正文格式照 §2.1（字數、連結、書名號、禁令句式 grep）。
  5. `python3 scripts/check_songs.py`＋`build_pages.py` 全綠。
- **完成判準**（一個時代頁算「完成」＝以下全過）：盲驗全 PASS → build 綠 → 部署 → `verify_live.py` 綠 → David 抽驗文字。缺任何一環都不准回報完成。
- **升級規則**：同一頁修兩輪還不過，停手回報 David，不試第三輪。

## 8. 不做清單（防隨性條款——做了任何一條＝該批打掉重做）

1. 不引用歌詞（紅線，再說一次）。
2. 不自創 schema 欄位、不改分期表、不增刪 era——框架問題回報 David 拍板，不自行「優化」。
3. 不動既有頁面（人物頁、field 頁、地圖、既有 tab 行為）；S3 之前不新開人物頁。
4. 不放任何圖片／音檔進 repo（全域 pre-commit hook 會擋；歌曲線 S1–S2 根本不需要圖）。
5. 不手寫歌單 HTML/MD——歌單只能從登記簿生成。
6. 不收侵權連結、不收歌單頁、不「先放著之後再換」。
7. 不加裝飾性 CSS／動效／icon；樣式沿用現有 field 頁，S1 基建定案後不再動樣式。
8. 不在正文塞滿歌名清單充字數——正文是敘事，清單交給歌單區。
